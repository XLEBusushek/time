# Work Track

Мини-приложение для тайм-менеджмента: **работы** (крупные задачи) и **этапы** внутри каждой с галочками прогресса. Данные хранятся в `localStorage` браузера.

## Как пользоваться

1. Откройте `index.html` в браузере (двойной клик или перетащить в окно).
2. Введите название работы и список этапов (каждый с новой строки), нажмите «Добавить работу».
3. Отмечайте выполненные этапы чекбоксами; при необходимости добавляйте этапы полем внизу карточки.

## Git и GitHub

```bash
cd work-track
git init
git add .
git commit -m "Initial: work tracker with stages"
```

На GitHub: **New repository** → без README (у вас уже есть локально), затем:

```bash
git remote add origin https://github.com/<ваш-логин>/work-track.git
git branch -M main
git push -u origin main
```

Опционально включите **Settings → Pages → Branch: main, folder / (root)** — получится публичная ссылка на тот же интерфейс.

## ИИ-советник (OpenAI через прокси)

Браузер не может напрямую вызывать OpenAI (CORS и ключ). В интерфейсе блок **«ИИ-советник»**: без URL прокси показываются только простые локальные подсказки; с прокси уходит снимок ваших работ/этапов и приходит текст рекомендаций.

### Cloudflare Worker (бесплатный тариф)

1. [Cloudflare Workers](https://developers.cloudflare.com/workers/) → создать Worker, вставить код из `worker/openai-proxy.js`.
2. **Settings → Variables** для Worker:
   - `OPENAI_API_KEY` — секрет с [platform.openai.com](https://platform.openai.com/);
   - опционально `PROXY_SECRET` — случайная строка; ту же строку вписать в приложении в поле «Секрет», чтобы посторонние не расходовали ваш ключ;
   - опционально `OPENAI_MODEL` (например `gpt-4o-mini`).
3. Сохранить Worker, скопировать его URL (`https://….workers.dev`), вставить в приложении → **Сохранить настройки** → **Рекомендации ИИ**.
